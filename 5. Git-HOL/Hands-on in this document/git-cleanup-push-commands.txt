Git Hands-On Lab 5 - Clean Up and Push to Remote
====================================================

1. Verify master is in a clean state:

git checkout master
git status

Expected:
nothing to commit, working tree clean


2. List all available branches:

git branch -a


3. Pull the remote Git repository to master:

git pull origin master

If your repository uses main instead of master:

git checkout main
git pull origin main


4. Push pending changes from the previous hands-on to the remote repository:

git push origin master

For a repository using main:

git push origin main


5. Verify the changes in the remote repository:

Open the remote GitHub/GitLab repository in your browser and confirm that
the latest commits and files are visible.

You can also verify the remote information locally:

git remote -v

Check the latest commits:

git log --oneline --graph --decorate --all

Check final status:

git status


OPTIONAL SAFE WORKFLOW
----------------------

Before pulling, confirm the current branch:

git branch --show-current

Fetch remote information:

git fetch origin

Check status:

git status

Pull:

git pull origin master

Push:

git push origin master


MAIN BRANCH NOTE
----------------

The hands-on document uses "master". Many newer repositories use "main".
If your repository uses main, replace master with main in the commands.

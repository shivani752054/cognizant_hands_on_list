Git Hands-On Lab 4 - Merge Conflict Resolution
===================================================

1. Verify master is clean:
git checkout master
git status

2. Create and switch to GitWork:
git branch GitWork
git checkout GitWork

# Or:
# git checkout -b GitWork

Create hello.xml:
echo '<message><content>Hello from GitWork branch</content></message>' > hello.xml

3. Update hello.xml and observe status:
echo '<message><content>Updated from GitWork branch</content></message>' > hello.xml
git status

4. Commit branch changes:
git add hello.xml
git commit -m "Update hello.xml in GitWork"

5. Switch to master:
git checkout master

6. Create hello.xml on master with DIFFERENT content:
echo '<message><content>Hello from master branch</content></message>' > hello.xml

7. Commit master changes:
git add hello.xml
git commit -m "Add hello.xml in master"

8. Observe complete log:
git log --oneline --graph --decorate --all

9. Check differences:
git diff master GitWork

10. View differences using configured P4Merge:
git difftool master GitWork

11. Merge GitWork into master:
git merge GitWork

Git should report a conflict in hello.xml.

12. Observe conflict markup:
cat hello.xml

The file may contain markers such as:

<<<<<<< HEAD
master content
=======
GitWork content
>>>>>>> GitWork

13. Resolve using a 3-way merge tool:
git mergetool

If P4Merge is configured as the merge tool, it can be used here.

Example configuration:
git config --global merge.tool p4merge
git config --global mergetool.p4merge.cmd '"C:/Program Files/Perforce/p4merge.exe" "$BASE" "$LOCAL" "$REMOTE" "$MERGED"'

The P4Merge path depends on your installation.

After resolving the conflict, verify hello.xml and stage it:
git add hello.xml

14. Commit resolved merge:
git commit -m "Resolve merge conflict between master and GitWork"

15. Observe status:
git status

Add backup files generated by merge tools to .gitignore:
echo "*.orig" >> .gitignore
echo "*.bak" >> .gitignore
echo "*~" >> .gitignore

16. Commit .gitignore:
git add .gitignore
git commit -m "Ignore merge backup files"

17. List branches:
git branch -a

18. Delete merged branch:
git branch -d GitWork

19. Observe final log:
git log --oneline --graph --decorate


MAIN BRANCH NOTE
----------------
The HOL uses "master". If your repository uses "main", replace master with main
in the commands above.
